HUONG DAN TRINH BAY KHI GIANG VIEN HOI VE CAC LUONG MA HOA/HASH TRONG DO AN
Ngay cap nhat: 2026-03-31

============================================================
0) Mo dau nen noi truoc (30-45s)
============================================================
Trong he thong nay co 3 lop bao mat chinh:
1. Bao mat du lieu luu tru (at rest): AES-256-GCM trong backend AesService.
2. Bao mat xac thuc bi mat (password/PIN): PBKDF2-HMAC-SHA256 voi salt ngau nhien + iteration cao.
3. Bao mat du lieu truyen app-layer (ngoai HTTPS): session key AES duoc trao doi bang RSA-OAEP, sau do payload API duoc ma hoa AES-GCM theo tung request.

Y quan trong de ghi diem:
- AES la block cipher 16-byte, nhung trong GCM/CTR no hoat dong nhu stream cipher nen ma hoa duoc payload dai bat ky.
- PBKDF2 trong code nay dung de hash password/PIN, KHONG dung de dan xuat AES master key.
- RSA trong code nay chu yeu de wrap session key, khong ma hoa payload lon.


============================================================
1) Luong ma hoa AES-256-GCM trong code nhu the nao? Vi du khi dang ky
============================================================

1.1 Duong di request dang ky tu frontend
- frontend/src/pages/RegisterPage.tsx
  ham handleSubmit() goi: api.post('/auth/register', body).
- frontend/src/api/client.ts
  request interceptor se tu dong goi prepareEnvelopeRequest(...), gan header:
  X-App-Envelope, X-App-Timestamp, X-App-Nonce, X-App-Session-Key.
  Neu body JSON thi body duoc ma hoa thanh envelope {iv, tag, payload, aad}.

Chuoi goi chinh (frontend):
RegisterPage.handleSubmit
-> api.post('/auth/register', ...)
-> client.ts request interceptor
-> prepareEnvelopeRequest() (transportEnvelope.ts)
-> rsaOaepEncryptSessionKey() + encryptJsonPayload(AES-GCM WebCrypto)

1.2 Backend nhan request dang ky
- backend/src/crypto/interceptors/transport-envelope.interceptor.ts
  Interceptor global (duoc gan APP_INTERCEPTOR trong crypto.module.ts) xu ly truoc controller:
  1) Kiem tra metadata x-app-*.
  2) build AAD = METHOD|PATH|TIMESTAMP|NONCE.
  3) rsaTransport.decryptSessionKey(...) de lay AES session key 32 bytes.
  4) decryptEnvelope(...) de giai ma body JSON.
  5) cho request vao AuthController/AuthService.

Chuoi goi chinh (backend inbound):
TransportEnvelopeInterceptor.intercept
-> RsaTransportService.decryptSessionKey
-> rsaOaepDecrypt (rsa-manual.ts)
-> RsaTransportService.decryptEnvelope
-> decryptGCM (pure TS AES-GCM)
-> AuthController.register
-> AuthService.register

1.3 Trong AuthService.register, AES dung cho du lieu nao
- backend/src/modules/auth/auth.service.ts
  trong register():
  + password duoc hash bang PBKDF2 (khong AES): pbkdf2.hashSecret(dto.password, 'password').
  + cac truong nhay cam customer duoc AES encrypt:
    phone, cccd, dateOfBirth, address (qua aesService.encrypt).
  + so tai khoan duoc encrypt qua AccountCryptoService.encryptAccountNumber(...).

1.4 Co che AES-GCM core (tai sao ma hoa duoc >16 byte)
- backend/src/crypto/aes-gcm/gcm.ts + aes-core.ts + ghash.ts
  encryptGCM() lam:
  1) expandKey256(key): mo rong key thanh 60 words.
  2) tao H = E(K, 0^128) cho GHASH.
  3) tao J0 tu IV 12 byte.
  4) CTR mode:
     voi moi block counter CB_i, tinh keystream_i = E(K, CB_i),
     ciphertext_i = plaintext_i XOR keystream_i,
     tang counter inc32 va lap lai den het payload.
  5) Tag = GHASH(H, AAD, C) XOR E(K, J0).

Giai ma:
- Kiem tra tag truoc (chong sua du lieu), hop le moi XOR nguoc CTR de lay plaintext.

1.5 Response dang ky tra ve frontend
- Van qua interceptor:
  map(data) -> rsaTransport.encryptWithSessionKey(payload, sessionKey, aad)
  -> response body la encrypted envelope, header x-app-envelope = 1.
- frontend client.ts response interceptor:
  maybeDecryptResponse() -> decryptEnvelopeResponse() bang sessionKey da luu trong config request.


============================================================
2) Luong hash mat khau/PIN voi PBKDF2 nhu the nao?
============================================================

2.1 File chinh
- backend/src/crypto/services/pbkdf2.service.ts

2.2 hashSecret(secret, purpose)
- purpose = 'password' thi iterations = 310000.
- purpose = 'pin' thi iterations = 220000.
- dkLen = 32 bytes.
- salt = randomBytes(16).
- derived = pbkdf2(secret, salt, iterations, 32).
- Luu chuoi format:
  pbkdf2$sha256$iterations$saltHex$derivedHex

2.3 verifySecret(secret, stored)
- Tach 5 thanh phan tu chuoi stored.
- Parse iterations, salt, expected hash.
- Tinh lai actual = pbkdf2(secret, salt, iterations, expected.length).
- So sanh timingSafeEqual(actual, expected) de giam timing attack.

2.4 Cac diem goi quan trong
- Dang ky user: AuthService.register -> pbkdf2.hashSecret(password, 'password').
- Dang nhap: AuthService.login -> pbkdf2.verifySecret(dto.password, user.passwordHash).
- PIN customer:
  + Tao PIN: CustomersService.createProfile/setupPin/setPin -> hashSecret(pin, 'pin').
  + Verify PIN: CustomersService.verifyPin -> verifySecret(pin, customer.pinHash).

2.5 Neu giang vien hoi PBKDF2 co dan xuat key ma hoa khong
- Tra loi dung theo code hien tai:
  KHONG.
- AES master key dang lay truc tiep tu env AES_MASTER_KEY trong AesService constructor.
- PBKDF2 hien tai chi dung cho password/PIN hashing va verify.


============================================================
3) Luong RSA cho AES key, OAEP lam gi, va trao doi frontend-backend
============================================================

3.1 Muc dich
- RSA khong ma hoa payload lon.
- RSA chi ma hoa (wrap) session key AES 32 bytes moi request.

3.2 Public key bootstrap
- backend/src/crypto/transport.controller.ts
  GET /transport/public-key -> RsaTransportService.getPublicKeyMeta()
  tra ve {alg, kid, n, e}.
- frontend/transportEnvelope.ts
  getTransportPublicKeyMeta() fetch endpoint tren va cache ngan han.

3.3 Frontend tao va ma hoa session key
- prepareEnvelopeRequest():
  1) Tao sessionKey random 32 byte.
  2) rsaOaepEncryptSessionKey(sessionKey, keyMeta).
  3) Gan vao header X-App-Session-Key (base64).
  4) Neu co body JSON: encryptJsonPayload(sessionKey, aad) thanh envelope.

3.4 Backend giai ma session key va body
- TransportEnvelopeInterceptor:
  1) Doc X-App-Session-Key.
  2) rsaTransport.decryptSessionKey(...).
  3) decryptEnvelope(..., expectedAad).
  4) req.body = JSON da giai ma.

3.5 OAEP la gi va OAEP lam nhung gi
- OAEP = Optimal Asymmetric Encryption Padding.
- Vai tro: bien RSA thanh xac suat (co random), chong deterministic-pattern attack.
- backend/src/crypto/pure/rsa-manual.ts:
  + mgf1(seed, len): sinh mask tu SHA-256.
  + rsaOaepEncrypt:
    lHash = SHA256(label rong),
    DB = lHash || PS || 0x01 || message,
    maskedDb = DB XOR MGF1(seed),
    maskedSeed = seed XOR MGF1(maskedDb),
    EM = 0x00 || maskedSeed || maskedDb,
    roi moi thuc hien RSA core c = m^e mod n.
  + rsaOaepDecrypt:
    m = c^d mod n,
    unmask seed/DB,
    verify lHash,
    tim marker 0x01,
    lay lai message.

3.6 Trao doi du lieu 2 chieu (de trinh bay ro)
- Chieu request:
  Frontend tao session key -> RSA-OAEP encrypt session key -> AES-GCM encrypt payload -> gui.
- Chieu response:
  Backend dung cung session key vua giai ma de encrypt response bang AES-GCM + AAD,
  frontend dung sessionKey cache trong request config de decrypt response.

4) Luong hien thi truoc PIN va sau PIN khac nhau the nao?
============================================================

4.1 Frontend logic
- DashboardPage.tsx:
  + GET /customers/me kem query viewToken neu co.
  + profile.isPinVerified quyet dinh UI dang "bao ve" hay "xem day du".
  + Nut "Xem chi tiet" mo PinModal.
- PinModal.tsx:
  + submit goi POST /customers/me/verify-pin {pin}.
  + neu thanh cong nhan {viewToken, ttlSeconds, expiresAt} -> setViewToken(viewToken).
  + query profile refetch voi viewToken.

4.2 Backend logic
- CustomersController.getMyProfile(viewToken?)
  -> CustomersService.getProfile(..., viewToken).
- CustomersService.getProfile:
  + pinMode = isOwner && isPinViewSessionValid(userId, viewToken).
  + neu pinMode true: MaskingEngine.mask(..., isPinVerified=true) cho CUSTOMER -> tra full value.
  + neu pinMode false: tra data da mask theo rule customer chua xac thuc.
  + tra ve isPinVerified = pinMode, hasPin.

4.3 Rule mask quan trong trong MaskingEngine (customer)
- role CUSTOMER + isPinVerified=true -> tra thang value (full).
- role CUSTOMER + chua verify:
  + phone/cccd: an giua.
  + email: an ky tu cuoi local-part.
  + balance: '......'.
  + cvv: '***'.
  + card_number: rut gon.
  + account_number: cho owner xem field rieng (khong an).

4.4 Co che token ngan han
- verifyPin thanh cong -> createPinViewSession(userId): token song 120 giay.
- Neu token het han/sai/thieu -> pinMode=false.
- updateProfile cung bat buoc token hop le, neu khong se bi Forbidden.


============================================================
5) Mau tra loi ngan gon de noi truoc giang vien (co the hoc thuoc)
============================================================
"He thong em tach 3 bai toan: ma hoa du lieu luu tru bang AES-256-GCM,
hash bi mat bang PBKDF2-HMAC-SHA256, va bao ve request/response app-layer
bang RSA-OAEP + AES session key.

Khi register, frontend goi /auth/register qua axios interceptor, he thong tu tao
session key, RSA-OAEP ma hoa session key, AES-GCM ma hoa body. Backend interceptor
se giai session key bang RSA private key, giai body bang AES-GCM roi moi vao AuthService.
Trong AuthService, password duoc PBKDF2 hash, con phone/cccd/dateOfBirth/address duoc AES encrypt
truoc khi luu DB.

PBKDF2 cua em dung salt random 16 bytes, password 310k rounds, PIN 220k rounds,
luu dinh dang pbkdf2$sha256$iter$salt$hash va verify bang timingSafeEqual.

Khi customer chua verify PIN, profile tra ve du lieu da mask; verify PIN thanh cong
se cap viewToken 120 giay. Trong khoang do, getProfile voi viewToken hop le se tra
isPinVerified=true va hien full field theo role CUSTOMER." 


============================================================
6) Cac diem de giang vien hay hoi them (nen chuan bi)
============================================================
1. Vi sao AES-GCM an toan hon CBC?
- Vi co tag xac thuc toan ven + AAD, chan sua doi ciphertext.

2. Vi sao RSA khong ma hoa payload lon?
- RSA cham va gioi han kich thuoc; dung de wrap key la dung mo hinh hybrid cryptography.

3. Vi sao can nonce/timestamp/AAD?
- Chong replay va rang buoc context request (method/path/time/nonce).

4. PBKDF2 co dung de sinh AES master key khong?
- Trong code nay khong; AES master key doc tu env.

5. Rui ro neu dung lai IV voi cung key trong GCM?
- Rat nguy hiem: ro thong tin va anh huong toan ven.


Ket luan:
Neu trinh bay theo thu tu "input -> interceptor -> service -> algorithm -> output"
va noi ro ham nao goi ham nao nhu tren, giang vien se thay minh hieu ca code flow
va ban chat thuat toan, khong chi hoc thuoc ly thuyet.
